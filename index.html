Late Deliverables and Manual Processes Threatened a Major Client Relationship

The firm provides market research and data science consulting for major entertainment studios and global brands. A Fortune 500 media conglomerate, one of the firm's largest clients, required survey data processed and delivered across 15 international markets on tight release cycles. Deliverables arrived late. The client relationship was deteriorating. No one had scoped a technical solution. I saw the bottleneck, pitched the fix, and built it.

System 1: A Multilingual NLP Engine Automated Qualitative Coding Across 14 Languages

The first system automated the most labor-intensive step in the pipeline: reading, categorizing, and coding thousands of open-ended survey responses per wave. The system ingests raw SPSS data, sanitizes responses in 14 languages, performs fuzzy string matching against standardized codeframes, enriches unresolved titles via the TMDB API, and outputs formatted, client-ready Excel deliverables.

Ingestion

Reads .sav files, preserves value labels and metadata, handles translation column variants, reshapes wide survey data into analysis-ready long format with automatic variable prefix detection.

Sanitize
14-Language Non-Answer Filtering
Built 14 country-specific non-answer dictionaries by hand, one per market. Handled Unicode normalization, vowel detection for Latin scripts, and special-case logic for Japanese characters (katakana, hiragana, kanji). No off-the-shelf NLP library covered this; each market's junk responses had to be cataloged and filtered individually.
unicodedata, regex, 14 custom dictionaries

TMDB API with Bilingual Fallback
Instead of waiting for an enterprise master data system to categorize TV shows (which could take months of procurement), I wired the public TMDB API directly into the pipeline. The system queries in the respondent's native language first, falls back to English, pulls metadata (year, network, popularity, streaming provider), filters against an allowed brand list, and auto-matches results back to the codeframe.
TMDB REST API, requests, brand filtering

System 2: A Concurrent Pipeline Reverse-Engineered Formatted Crosstabs into Queryable Databases

The second system solved the opposite problem: converting presentation-formatted Excel crosstabs back into queryable databases. These crosstabs were designed for human reading, not machine ingestion. Statistical significance was encoded as cell background colors, not as data fields. A peacetime approach would have filed a ticket with the data provider requesting clean JSON. I wrote a script to rip the hex values out of the cells instead.

Color-Based Data Extraction
Programmed openpyxl to physically read cell background hex values: FFFF0000 for statistically down, FF00FF00 for statistically up. Reverse-engineered visual formatting into structured data columns because the upstream provider had no API and no clean export. The data had to come from somewhere.

The Real Win Was Not Automation. It Was Shifting the Team from Linear to Iterative Workflows

On the surface, this is an automation story: scripts replaced manual work and cut reporting time by 99%. The deeper win was an operating model shift. Before the automation, the firm ran a linear workflow. Every quarter started from zero: client specs came in, analysts began processing, and the team raced the clock sequentially. My systems made iterative workflows possible. Directors could negotiate specs with the client while analysts pre-wrote queries, updated codeframes, and staged quarter-over-quarter adjustments in parallel. By the time data dropped, 80% of the work was already loaded. Go time became execution, not discovery.

06

The Automation Inverted the Team's Error Philosophy: Large Visible Threats Over Small Hidden Ones

The old process optimized for preventing small mistakes at every step: hand-check every row, verify every categorization, move slowly. The result was dozens of small, invisible errors scattered across the pipeline, each one impossible to catch until it surfaced in a client deliverable. The automation inverted this. Run the full pipeline fast. Produce large, obvious mismatches (a whole market miscoded, a codeframe off by 500 rows) that are immediately visible in the output. Then fix them in the next iteration. Small errors hide. Large errors announce themselves. The team stopped tiptoeing through a minefield and started eliminating concentrated threats one at a time.
